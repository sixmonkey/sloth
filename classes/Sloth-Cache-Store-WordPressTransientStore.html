WordPress Transients Cache Store.

Implements the Laravel Cache Store contract using WordPress transients, allowing themes that prefer not to use filesystem caching to store cache data in the WordPress database via the Transients API.

Usage

Use via the Cache facade with the 'wp-transients' driver:

Cache::driver('wp-transients')->rememberForever('key', fn() => expensive());

Or set as default in CacheServiceProvider if preferred over file cache.

Prefix

All keys are prefixed with 'sloth_' to avoid collisions with other plugins that use WordPress transients.

Expiry

WordPress transients use seconds for expiry. Passing 0 stores the value indefinitely (no expiry). This maps to Laravel's forever() behaviour.
